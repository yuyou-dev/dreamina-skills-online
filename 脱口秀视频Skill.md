---
技能名称: 脱口秀视频Skill
技能描述: 一句话生成脱口秀视频的即梦在线版复合技能。用户上传脱口秀角色图片、角色海报、人物设定图或短视频参考后，只需说一个主题，即可自动分析角色形象与表演风格，自动写脱口秀脚本，先让用户确认脚本，再默认使用 seedance2.0fast_vip 模型分段生成视频素材，最后用 video_editor 拼接为完整脱口秀视频输出。当用户说“用这个角色做脱口秀”“一句话生成脱口秀视频”“上传角色生成单口喜剧”“帮我写稿并生成脱口秀短视频”时使用。
---

技能内容：

本技能是**脚本确认型视频复合 skill**。核心目标是：用户上传一个脱口秀角色，再输入一句主题或吐槽方向，技能自动完成角色分析、段子脚本、分镜分段、Seedance 视频生成和最终拼接。

即梦在线版链式调用容易停在单步结果，因此本技能必须显式写出每一步的下一步工具。唯一默认需要用户参与确认的是“脚本确认”；脚本确认后，后续分段生成和拼接应自动继续。

本技能必须独立完成批量、链式调用和并发规划；不要假定即梦在线版能读取外部 Markdown。脱口秀任务的数量单位是 `segment_id`：脚本确认后有几段，就必须生成几段视频素材，并最终拼接成 1 条完整视频。执行时先在本技能内生成分段 manifest，再按依赖分批生成和拼接。

## 一、触发范围

### 1.1 应使用本技能的情况

- 用户上传角色图、人物设定图、角色海报、虚拟主播图、玩偶形象、IP 角色图或短视频片段。
- 用户要求生成脱口秀、单口喜剧、stand-up comedy、吐槽视频、角色口播段子、舞台表演短视频。
- 用户只给一句主题，例如“让这个角色吐槽打工人周一”“用这个角色讲一段 AI 焦虑脱口秀”。
- 用户希望自动写脚本、确认脚本后生成完整视频。

### 1.2 不应使用本技能的情况

- 用户只想写纯文本段子，不需要视频。
- 用户没有上传角色，也没有给出可生成角色的描述；此时先收集角色信息。
- 用户要求冒充真实人物、伪造真实人物发言、制造误导性新闻或攻击现实个人。
- 用户要求仇恨、骚扰、露骨色情、危险违法、未成年人不当内容等不适合生成的表演。

## 二、即梦在线工具映射

| 工具名 | 本技能用途 | 链式调用提示 |
|---|---|---|
| `generate_form_for_info_collection` | 缺少角色素材、主题、视频时长等必填信息时收集 | 表单完成后立即进入角色分析和脚本生成 |
| `creation_agent_search` | 可选：搜索热点背景、行业梗、主题参考 | 搜索结果只作为段子素材，不编造事实 |
| `image2image` / `image2image_v3` | 基于上传角色生成舞台关键帧、角色口播参考帧、封面图 | 关键帧生成后进入视频分段 |
| `text2image` / `text2image_v3` | 用户无图但给了角色描述时生成角色母图 | 母图通过后作为角色身份参考 |
| `image2video` | 单张角色舞台图生成一个简短表演片段 | 适合单段 4-6 秒片段 |
| `multi_frame2video` | 多张关键帧串成一段连续表演 | 适合开场/转场/结尾 |
| `multi_modal2video` | 默认视频生成工具，使用角色图、舞台图、脚本、口型/动作提示生成分段素材 | 默认使用 `seedance2.0fast_vip` |
| `start_end2video` | 明确控制开场到包袱反应、站位变化、镜头推进 | 可选，用于关键转折 |
| `video_editor` | 拼接所有分段视频，加入字幕、节奏停顿、音效或背景音乐 | 视频流程最后必须调用 |
| `image_super_resolution` | 高清化封面、角色关键帧或舞台图 | 只对身份正确的图使用 |

在线版适配：

- 默认视频模型写为 `seedance2.0fast_vip`。
- 若在线环境模型列表使用不同展示名，以最接近 Seedance 2.0 Fast VIP / 高质量快速视频的选项为准。
- 图片工具如提供 v3，优先使用 `image2image_v3` / `text2image_v3`。
- 不写本地 CLI 登录、任务 ID 查询或下载命令。

## 三、输入字段

| 字段 | 必填 | 缺失时处理 |
|---|---|---|
| 角色素材 | 是 | 未上传图片/视频且无角色描述时，调用 `generate_form_for_info_collection`，提示上传或描述角色 |
| 脱口秀主题 | 是 | 缺失时提示用户给一句主题，例如“吐槽打工人周一” |
| 视频时长 | 否 | 默认 30 秒；短视频可 15 秒，完整段落可 60-90 秒 |
| 语言 | 否 | 默认中文 |
| 风格 | 否 | 默认轻松犀利、节奏紧凑、适合短视频 |
| 舞台场景 | 否 | 默认小剧场单口喜剧舞台，暖色聚光灯，观众区虚化 |
| 角色声音 / 口播 | 否 | 默认生成字幕脚本；如在线工具支持音频/配音，可加入多模态参考 |
| 输出比例 | 否 | 默认 9:16；横版舞台可用 16:9 |
| 分段长度 | 否 | 默认每段 4-6 秒 |
| 是否先确认脚本 | 否 | 默认必须确认脚本后再生成视频 |

## 四、角色分析

收到角色素材后，先自动分析，不向用户展示过长技术描述。

分析维度：

- 角色身份：人类、虚拟角色、动物拟人、玩偶、机器人、品牌吉祥物等。
- 外观锁定：发型、服装、配色、脸部特征、道具、体型、标志性元素。
- 表演气质：冷幽默、夸张、毒舌、社畜感、知识分子、可爱荒诞、反差萌。
- 舞台适配：站立麦克风、手持麦、吧台、小剧场、开放麦、综艺棚。
- 禁止变化：不得换脸、换服装主色、改标志性道具、添加无关角色。

角色身份锁模板：

```text
角色身份锁:
- 角色类型:
- 外观特征:
- 服装/道具:
- 表情气质:
- 舞台设定:
- 禁止变化:
```

## 五、脚本生成与确认

脚本确认是本技能的关键交互阶段。默认必须先让用户确认脚本，再进入视频生成。

### 5.1 脱口秀脚本结构

默认 30 秒脚本：

| 段落 | 时长 | 内容 |
|---|---|---|
| Hook | 0-4 秒 | 一句强开场，点题并制造反差 |
| Setup | 4-10 秒 | 讲清处境，建立共鸣 |
| Build | 10-18 秒 | 连续 2-3 个递进吐槽 |
| Punchline | 18-25 秒 | 主包袱，语义反转或夸张落点 |
| Tag / Ending | 25-30 秒 | 追加小包袱或角色招牌收尾 |

脚本要求：

- 每句适合口播，不写书面长句。
- 每 4-6 秒自然切一段，方便视频分段生成。
- 包袱要有铺垫和落点，避免只写网络热词。
- 保持角色口吻，不能变成普通旁白。
- 避免攻击现实个人、群体歧视、虚假事实和危险建议。

### 5.2 脚本确认输出

在生成视频前，必须先给用户确认稿：

```Markdown
# 脱口秀脚本确认

## 角色理解
- 角色:
- 表演气质:
- 舞台:

## 脚本
| 分段 | 时间 | 台词 | 表演动作 | 镜头 |
|---|---|---|---|---|

## 视频生成设置
- 模型: seedance2.0fast_vip
- 比例:
- 分段长度:
- 预计段数:

请确认脚本。确认后我会按分段生成视频素材，并用 video_editor 拼接成完整脱口秀视频。
```

用户确认方式：

- “确认”“可以”“生成视频”“就按这个来” → 进入执行阶段。
- 用户修改某句台词 → 只改对应分段，再重新确认。
- 用户要求换风格 → 重写脚本，再确认。

## 六、视频分段生成

脚本确认后进入执行阶段，不再停下询问。

### 6.1 分段规划

把脚本拆成 4-6 秒一段。每段必须有：

- segment_id，例如 `S01`。
- 台词。
- 表情。
- 身体动作。
- 镜头运动。
- 舞台灯光。
- 角色身份锁。
- 与上一段的衔接。

默认 30 秒视频建议 6 段：

| 段落 | 时长 | 画面目标 |
|---|---|---|
| S01 | 0-5 秒 | 角色登场，开场 Hook |
| S02 | 5-10 秒 | 建立吐槽处境 |
| S03 | 10-15 秒 | 第一个递进笑点 |
| S04 | 15-20 秒 | 第二个递进笑点 |
| S05 | 20-25 秒 | 主包袱和观众反应 |
| S06 | 25-30 秒 | 结尾 tag 和收场 |

### 6.1.1 分段任务清单和数量守恒

脚本确认后立即生成 manifest：

```Markdown
| job_id | target | tool | depends_on | status | quality_gate |
|---|---|---|---|---|---|
| TS-KEYFRAME | 舞台关键帧 | image2image_v3 | 角色素材 | planned | 角色身份锁 |
| TS-S01 | Hook 分段视频 | multi_modal2video | TS-KEYFRAME, 脚本确认 | planned | TS-V1/TS-V3 |
| TS-S02 | Setup 分段视频 | multi_modal2video | TS-KEYFRAME, 脚本确认 | planned | TS-V1/TS-V3 |
| TS-FINAL | 完整脱口秀视频 | video_editor | TS-S01...TS-S06 | planned | TS-V5 |
```

计数规则：

- `requested_count = confirmed_segment_count + 1 final_video`。
- `planned_count` 必须等于所有分段任务 + 最终 `video_editor` 任务。
- 任何分段失败都必须保留原 `TS-Sxx` 编号重试。
- 如果某个分段失败且未修复，`TS-FINAL` 不得标记完成。

### 6.1.2 并发规则

- 脚本未确认前，不生成任何视频分段。
- 角色关键帧未通过前，不并发生成所有分段。
- 脚本确认且关键帧通过后，`TS-S01` 到 `TS-Sxx` 可按 2-4 个并发生成。
- 需要强连续性的开场/结尾可单独生成；普通中段可并发。
- `video_editor` 必须等待所有分段进入 `done`，并按脚本顺序拼接。
- 如果并发导致角色漂移，下调并发，增加每段的角色身份锁和上段衔接描述。

### 6.2 默认生成工具

默认使用：

```text
工具: multi_modal2video
模型: seedance2.0fast_vip
输入:
- @图片1: 用户上传角色图或生成的角色母图
- @图片2: 舞台/封面关键帧（如已生成）
- 当前分段台词
- 表情、动作、镜头、灯光提示
输出:
- 当前 segment_id 对应的视频素材
```

如果只有一张角色图且动作简单，也可使用 `image2video` 生成单段素材；但完整脱口秀默认仍优先 `multi_modal2video`，因为它更适合把角色、台词、舞台、动作和多段一致性一起控制。

### 6.3 分段提示词模板

```text
使用 @图片1 作为角色身份参考，生成脱口秀视频分段 {segment_id}。
模型设置: seedance2.0fast_vip。
画幅: {ratio}。
时长: {duration} 秒。
角色身份锁: {character_lock}
舞台: 小剧场单口喜剧舞台，暖色聚光灯，观众区虚化。
台词: {line}
表演: {gesture}，表情 {expression}，节奏像真实脱口秀演员，保留自然停顿。
镜头: {camera}，角色始终清晰，口型尽量贴合台词节奏。
衔接: 延续上一段的站位、服装、灯光和角色外观。
禁止: 换脸、换服装主色、改变角色标志性元素、添加无关角色、字幕乱码、水印、夸张变形。
```

### 6.4 关键帧补充

如果上传角色图不适合直接生成视频，先调用 `image2image_v3`：

```text
基于用户上传角色图，生成脱口秀舞台关键帧。
保持角色外观、服装、道具和气质不变。
场景为小剧场单口喜剧舞台，角色站在麦克风前，暖色聚光灯，观众区虚化。
禁止改变角色身份、添加第二个主持人、添加水印和文字。
```

关键帧通过后，再作为 `@图片2` 输入 `multi_modal2video`。

## 七、拼接成片

所有分段生成后必须调用 `video_editor`。

```text
工具: video_editor
输入: S01-S06 分段视频素材
目标:
- 按脚本顺序拼接
- 保留 0.2-0.5 秒自然停顿
- 加入轻微现场氛围音或掌声笑声（如工具支持）
- 加字幕或保留字幕空间
- 输出完整脱口秀视频
```

不得把分段素材列表当作最终交付。最终交付必须是一个完整拼接视频，分段素材只作为过程产物。

## 八、质量检查

### 8.1 脚本质量

| 编号 | 标准 |
|---|---|
| TS-S1 | 有明确主题、开场、铺垫、递进、包袱、收尾 |
| TS-S2 | 台词口语化，适合角色说出口 |
| TS-S3 | 每段 4-6 秒，便于视频生成 |
| TS-S4 | 不攻击现实个人，不制造虚假事实，不含歧视和危险内容 |

### 8.2 视频质量

| 编号 | 标准 |
|---|---|
| TS-V1 | 角色外观、服装、道具和气质稳定 |
| TS-V2 | 每段都像舞台脱口秀表演，不是普通旁白 MV |
| TS-V3 | 口型、表情、动作与台词节奏尽量匹配 |
| TS-V4 | 分段之间灯光、站位、服装、舞台连续 |
| TS-V5 | 已用 `video_editor` 拼接成完整视频 |

失败处理：

- 角色漂移：强化角色身份锁，使用更多角色参考帧。
- 口型不稳：缩短单段台词，减少复杂动作。
- 分段断裂：用 `start_end2video` 或更明确的衔接描述修复关键段。
- 脚本不好笑：先重写脚本并再次确认，不直接生成视频。
- 视频未拼接：必须继续调用 `video_editor`。

## 十一、扩展预设空间

未来新增脱口秀预设时，不要假定即梦在线版会自动读取目录。应把新预设直接复制进当前 skill，或另建一个完整自包含的独立 skill。命名建议：

- `standup-office-worker.md`
- `standup-ai-anxiety.md`
- `standup-brand-mascot.md`
- `standup-live-commerce-host.md`
- `standup-knowledge-comedy.md`

每个预设应包含：角色类型、笑点结构、默认时长、分段数、动作库、镜头语言、禁用内容和 `video_editor` 拼接规则。
