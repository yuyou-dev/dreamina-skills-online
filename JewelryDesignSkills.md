---
技能名称: JewelryDesignSkills
技能描述: 专业珠宝设计与批量化出款技能。面向戒指、项链、耳环、手链、吊坠、胸针和高级珠宝系列设计，支持从简单描述、风格参考、产品图、草图或主题关键词出发，高效率生成真实可信、好看、可延展的珠宝设计图与设计说明。适合单款精品设计、批量设计、系列化方案、参考图变体、材质替换、风格矩阵和设计评审。当用户说“设计一款珠宝”“批量出款”“做 20 款戒指”“参考这个风格延展”“要真实好看又高效”时使用。
---

技能内容：

分为**交互阶段**和**执行阶段**。交互阶段提取珠宝类型、设计目标、数量、参考图和风格方向；执行阶段生成设计矩阵、调用图片工具、进行质量检查，并按数量要求完成单款或批量设计。

链式调用要求：每个阶段必须显式写出下一步，默认链路为“设计 brief → 设计矩阵 → 图片生成 → 质量检查 → 失败重试 → 最终设计表”。对于批量设计，不要让用户逐个确认 ID；应按用户给定数量直接组织批量生成。

批量执行要求：先生成设计矩阵和任务清单，再按批次并发生成。设计任务的数量单位是 `design_id`，用户要求 N 款就必须创建 N 个设计编号，并为每个编号生成图片或记录失败。

## 一、触发范围

### 1.1 应使用本技能的情况

- 用户要求设计戒指、项链、耳环、手链、吊坠、胸针、珠宝套装或高级珠宝系列。
- 用户要求“批量出款”“设计 N 款”“做系列延展”“参考某风格做变化”。
- 用户上传草图、产品图或参考图，希望做原创化变体。
- 用户更关心设计质量、真实结构、效率和设计流程，而不是上架图文视频。

### 1.2 应移交电商技能的情况

当用户已经选定设计，并继续要求：

- “做亚马逊九图”
- “生成商品图文案视频”
- “上架素材”
- “详情页”
- “投放短视频”

则输出已确认设计图、产品身份锁、材质宝石说明和可用于后续商品图生成的提示词。

## 二、工具映射

| 工具名 | 本技能用途 | 链式调用提示 |
|---|---|---|
| `generate_form_for_info_collection` | 只在缺少设计对象或数量时收集必填项 | 表单完成后进入设计矩阵，不停在问卷 |
| `creation_agent_search` | 查找非侵权风格、材料趋势、主题参考 | 搜索后转成原创设计约束 |
| `text2image` / `text2image_v3` | 从文字 brief 生成原创设计图 | 批量时按设计矩阵逐项生成 |
| `image2image` / `image2image_v3` | 基于草图、参考图、已有设计做变体 | 必须说明保留元素和可变元素 |
| `image_super_resolution` | 放大优秀设计图、细节图 | 结构正确后再超分 |
| `foreground_segmentation` | 抽取设计主体，用于对比板或后续电商素材 | 可选 |
| `image2video` / `multi_modal2video` | 只在需要设计展示短片时使用 | 默认使用 `seedance2.0fast_vip`；非主流程，电商视频交给电商技能 |

在线版适配：

- 图片工具如存在 v3 版本，优先使用 `text2image_v3` / `image2image_v3`。
- 图片类任务默认使用 4.7 模型的 2k 版。
- 视频类任务默认使用 `seedance2.0fast_vip`。
- 如用户明确指定图片或视频模型版本，按用户指定执行，不再套用对应默认模型。
- 不写本地 CLI 模型名、账号命令或任务查询命令。
- 运行时优先选择“产品设计”“商业摄影”“高清细节”“参考图改造”类在线预设。

| 字段 | 必填 | 缺失时处理 |
|---|---|---|
| 珠宝类型 | 是 | 缺失时调用 `generate_form_for_info_collection` |
| 设计目标 / 简单描述 | 是 | 缺失时要求补一句风格、材质、主题或用途 |
| 数量 | 否 | 默认 1 款；用户写 N 款时 N 是承诺数量 |
| 参考图 / 草图 | 否 | 有图则使用 `image2image` 做保留与变体 |
| 风格方向 | 否 | 默认高端真实珠宝设计 |
| 材质与宝石 | 否 | 从描述推断；推断不出时按类型默认 |
| 设计预设 | 否 | 默认 `single_refined_design` |
| 输出形式 | 否 | 默认每款 1 张主设计图 + 设计说明 |
| 图片模型 | 否 | 默认 4.7 模型的 2k 版；用户指定模型版本时按指定 |
| 视频模型 | 否 | 默认 `seedance2.0fast_vip`；用户指定模型版本时按指定 |
| 批量策略 | 否 | 默认先生成设计矩阵，再连续生成图片 |
| 输入素材类型 | 否 | 裸石 / 元素图 / 草稿 / 已有珠宝图；默认按上传内容判断 |
| 主石保留规则 | 否 | 默认保留主石颜色、琢型、比例和视觉重点 |
| 品类配比 | 否 | 批量设计默认混合戒指、耳饰、项链/吊坠、手链，不全部做成同一品类 |
| 构图密度 | 否 | 默认饱满但不杂乱；小轻金和日式轻古董可降低密度 |
| 宝石组合语言 | 否 | 默认用不同琢型副石建立节奏、层次、流动和边缘强调 |
| 画面表现 | 否 | 默认白底单件珠宝产品图，居中构图，柔和棚拍灯光 |
| 负向限制 | 否 | 默认禁止结构不合理、廉价饰品感、文字、logo、价格标签 |

## 四、设计身份与真实结构

每个设计必须定义：

- 类型：戒指、项链、耳环、手链、吊坠、胸针等。
- 轮廓：圆形、椭圆、羽毛、花卉、几何、流线等。
- 主视觉：主石、中心图案、链坠、耳坠、开口结构。
- 材质：金属色、表面工艺、珐琅、珍珠、彩宝等。
- 宝石方案：主石、配石、切工、颜色、排列方式。
- 结构：爪镶、包镶、密镶、链条、扣具、耳针、铰链、戒托、桥位。
- 佩戴逻辑：舒适度、重量感、开合方式、稳定性。

禁止：

- 大宝石没有支撑。
- 项链吊坠漂浮，没有吊环或连接结构。
- 手镯没有开口、铰链或弹性逻辑。
- 耳环过重但没有合理固定方式。
- 未经用户证明就声明克拉数、证书、产地或价格。

## 四点五、高质量图片提示词补充字段

每个生成提示词都应尽量包含以下字段。字段越完整，图片越容易稳定、真实、可控。

| 字段 | 写法要求 |
|---|---|
| `input_source` | 说明输入是裸石、元素图、草稿还是已有珠宝图 |
| `locked_reference` | 对输入素材必须保留的内容：主石颜色、琢型、比例、轮廓、构图或核心元素 |
| `category_mix` | 批量时写清品类分布，例如 10 款中戒指 3-4、耳饰 3-4、项链/吊坠 2-3 |
| `design_density` | 饱满华丽 / 轻盈克制 / 细密古董感；同时说明不要空、薄、散或杂乱 |
| `main_stone_role` | 一颗主石、双主石、主石与大副石组合、围绕主石展开 |
| `secondary_stone_plan` | 副石琢型、排列方式、节奏、层次、边缘强调 |
| `metal_and_finish` | 铂金、18K 白金、18K 黄金、玫瑰金、香槟金；高抛光、柔和磨砂、细腻金属反光 |
| `setting_language` | 爪镶、密镶、轨道镶、包镶、细爪镶、滚珠边、micro pavé 等 |
| `silhouette_logic` | 环绕式、半包围式、双石式、花簇式、流线式、镜面对称、轻微不对称 |
| `wearability` | 说明戒臂、链条、耳针、扣具、铰链、吊环等真实可佩戴结构 |
| `render_style` | 白底单件产品图，专业珠宝产品摄影或高精度珠宝渲染，柔和棚拍灯光 |
| `negative_rules` | 禁止项要具体，不只写“不要丑”；写清空心、过薄、结构散、概念化、卡通、文字 logo 等 |

### 4.5.0 专业提示词构建规则

每次调用图片工具前，先把用户输入整理成 `prompt_spec`。不要只输出风格形容词，要把保留项、可变项、工艺和成图规则写完整。

| 字段 | 写法要求 |
|---|---|
| `task_goal` | 原创设计、参考图变体、草图渲染、裸石成品化、风格矩阵或电商标准化 |
| `input_role` | 判断输入是裸石、草稿、已有珠宝、元素图、材质参考还是风格参考 |
| `preserve` | 明确不可改变的颜色、琢型、比例、轮廓、构图重心、结构或产品身份 |
| `change_scope` | 明确允许变化的镶嵌、金属结构、副石、表面工艺、边缘节奏或装饰语言 |
| `design_translation` | 把输入信息转成珠宝语言，例如主石布局、环绕结构、纹样转译、镂空、浮雕、宝石排列 |
| `construction` | 写清可佩戴结构：戒臂、链条、吊环、耳针、扣具、铰链、爪镶、包镶、桥位 |
| `material_craft` | 写清金属色、宝石、表面质感、工艺密度和细节尺度 |
| `rendering` | 款式设计默认白底或浅灰白底，单件居中，柔和棚拍光，清晰轮廓，轻微自然投影 |
| `negative_constraints` | 明确禁止文字、logo、水印、证书、价格、复杂背景、漂浮结构、塑料感、卡通感和不可生产结构 |

默认背景规则：

- 款式设计、参考图变体、草图渲染和裸石成品化默认输出白底图。
- 只有用户明确要求电商场景、模特佩戴、包装、生活方式、广告海报或平台背景时，才使用非白底背景。
- 如果输入图是黑底、手持、桌面或复杂环境，只把主体信息作为设计参考；输出仍按白底单件产品图执行。

### 4.5.1 输入素材处理

| 输入类型 | 处理方式 |
|---|---|
| 裸石 | 作为主石或核心视觉焦点，保留颜色、琢型、比例、透明度和视觉重心 |
| 元素图 | 提取元素轮廓、节奏和气质，转化为珠宝结构，不直接做平面图案 |
| 草稿 | 保留大致构图和主次关系，优化成可佩戴、可生产、结构完整的珠宝 |
| 已有珠宝图 | 保留可指定的轮廓、材质、主石和结构，变体时明确可变与不可变部分 |

### 4.5.2 宝石琢型与排列词库

常用琢型：

- 梨形、马眼形、椭圆形、祖母绿形、枕形、圆钻、长方形钻、梯形钻、公主方钻、榄尖形、蛋面、珠形珍珠。

排列方式：

- 环绕式、半包围式、放射式、花簇式、边缘强调式、阶梯式、镜面对称式、轻微不对称式、流线式、聚拢式、渐变式。

视觉目标：

- 建立节奏感、层次感、流动感、聚合关系、视觉中心、闪耀边缘、饱满轮廓。

### 4.5.3 画面表现通用规则

- 白色或极浅灰白背景。
- 单件珠宝居中展示。
- 专业珠宝产品摄影或高精度珠宝渲染。
- 柔和棚拍灯光或柔和自然光。
- 金属真实反光，宝石火彩清晰。
- 画面干净，无手、无人像、无文字、无 logo、无价格标签、无复杂道具。

### 4.5.4 黄金工艺提示词规则

| 工艺 | 正向描述 | 必须避免 |
|---|---|---|
| 黄金花丝 | 极细金丝、卷曲纹、花卉纹、蕾丝般镂空、轻盈通透、层次深度、手工节点；全花丝可占主体 70-90%，混合花丝占 10-30% | 平面印花、厚重实心堆砌、廉价电镀、镜面刺眼反光 |
| 錾刻浮雕 | 手工錾刻、浅浮雕、纹样从金属表面隆起、柔和压实边缘、细密刻线、自然手作痕迹 | 平面刻线、黑脏氧化、灰金、白金、铜色、锐利机械切割感 |
| 古法黄金 | 高纯度 24K 足金视觉、正暖黄色、温润哑光、细腻磨砂、柔和漫反射、边缘微圆润 | 橙铜色、18K 浅金、镜面抛光、发灰发白、电镀感、塑料感 |
| 3D 硬金 | 电铸硬金、圆润饱满、轻盈空心视觉、大圆角、柔和浮雕、哑光喷砂、细微闪点 | 尖锐棱角、沉重实心、灰白金属、粗糙卡通玩具感 |

### 4.5.5 镶嵌与复古风格规则

| 风格 | 设计重点 | 负向限制 |
|---|---|---|
| 商业高珠 | 主石为视觉中心，副石围绕主石形成饱满、聚拢、完整轮廓；梨形、马眼、椭圆、祖母绿、枕形、圆钻、长方形、梯形等琢型建立节奏和层次 | 空、薄、散、怪；空心大圆环；过大留白；概念装置；过度具象图案 |
| 日式轻古董 | 小体量、纤细、花园感、童话感；milgrain、beaded edge、micro pavé、细爪镶、镂空、微型花瓣和对称节奏 | 厚重粗大、过度满钻、赛博、塑料、卡通、黑底、人物佩戴图 |
| 小轻金日常 | 简约、轻盈、女性化；14K/18K 浅金、玫瑰金或暖黄金；少量小钻、彩宝或珍珠点缀 | 复杂堆砌、厚重结构、大面积满钻、珐琅、夸张卡通感 |
| 意大利复古 | 文艺复兴装饰、宫廷感、金属蕾丝、主石突出、绳纹边框、珠边、雕刻包边、拉丝缎光金属 | 现代极简、廉价饰品、塑料感、粗糙 3D、完全复制现有款 |

## 五、设计预设

### 5.1 `single_refined_design`

适合单款高质量设计。

输出：

- 1 张主设计图。
- 1 张可选细节图。
- 设计说明：名称、类型、材质、宝石、结构、佩戴场景、风险。

链路：

`text2image` / `image2image` → 质量检查 → `image_super_resolution`

### 5.2 `batch_design_set`

适合“设计 N 款”“批量出款”。

规则：

- N 是承诺数量，不要求用户先选 ID。
- 先生成设计矩阵，再按矩阵逐项调用图片工具。
- 每款必须有不同设计点，不能只换颜色。
- 每款至少输出 1 张图和 1 条设计说明。
- `requested_count = N`，`planned_count` 必须等于 N。
- 失败项保留原 `design_id` 重试，不用新编号替换。
- 不能用 1 张合集图、九宫格或 moodboard 代替 N 个设计结果，除非用户明确只要合集预览。

设计矩阵字段：

| 字段 | 说明 |
|---|---|
| design_id | 稳定编号，如 R01、R02 |
| name | 设计名 |
| type | 珠宝类型 |
| silhouette | 轮廓 |
| metal | 金属 |
| gem_plan | 宝石方案 |
| construction | 结构 |
| visual_hook | 视觉亮点 |
| prompt | 图片提示词 |
| model | 图片模型；默认 4.7 模型 2k 版 |

链路：

`generate_form_for_info_collection`（仅缺必填时） → 设计矩阵 → `text2image` / `image2image` 批量生成 → 质量检查 → 失败项重试

并发建议：

- 先生成完整设计矩阵，确认数量和差异策略。
- 第 1 批先跑 4-6 个 design_id，用于验证风格、结构和提示词强度。
- 第 1 批通过后，后续批次按 6-10 个 design_id 并发生成。
- 同一 design_id 的主图、细节图、材质变体存在依赖时，先主图后细节，不跨依赖并发。
- 质量检查和失败重试可以与下一批规划并行，但最终汇总必须等所有 design_id 有终态。

### 5.3 `high_jewelry_direction`

适合高级珠宝、秀款、艺术珠宝。

强调：

- 强主题：羽毛、花卉、星空、海洋、建筑、动物、东方纹样等。
- 工艺可信：隐藏结构、宝石支撑、可佩戴逻辑。
- 视觉高级：不靠 Logo，不靠假证书。

链路：

`creation_agent_search`（需要主题参考时） → 原创设计约束 → `text2image` → 细节图 → `image_super_resolution`

### 5.4 `reference_variation`

适合用户上传参考图后做原创变体。

保留：

- 大轮廓或佩戴类型。
- 主题情绪。
- 部分材质/宝石方向。

可变：

- 宝石排列。
- 金属结构。
- 轮廓比例。
- 工艺细节。

链路：

`image2image` → 变体 1/2/3 → 质量检查 → 选择优秀方向继续扩展

### 5.5 `material_gem_matrix`

适合同一设计做材质/宝石矩阵。

示例矩阵：

- 铂金 + 蓝宝石 + 钻石。
- 18K 黄金 + 祖母绿 + 钻石。
- 玫瑰金 + 粉色蓝宝石 + 珍珠。

链路：

基础设计图 → `image2image` 多版本材质替换 → 产品身份差异记录

### 5.6 `collection_series`

适合系列化方案。

输出：

- 系列主题。
- 3-12 款同系列设计。
- 每款共享 DNA，但类型、比例、宝石节奏不同。

链路：

```text
禁止: 不符合物理常识、大体量、厚重、粗戒臂、夸张奢华、过度满钻、现代极简工业风、赛博感、塑料感、廉价饰品感、卡通感、过度复杂不可佩戴结构、过大主石、粗糙金属、杂乱背景、手部佩戴图、人物、黑底、彩色背景、文字、logo、价格标签。
```

## 八、质量检查

| 编号 | 标准 |
|---|---|
| JD-1 | 设计符合用户要求的类型、风格和数量 |
| JD-2 | 图片真实好看，金属、宝石、比例可信 |
| JD-3 | 结构可佩戴，大宝石有支撑，链/扣/耳针/戒托逻辑存在 |
| JD-4 | 批量设计之间有明确差异，不只是颜色替换 |
| JD-5 | 参考图变体不过度复制原图 |
| JD-6 | 不编造证书、产地、克拉数、价格和品牌授权 |
| JD-7 | 有输入裸石或参考图时，主石颜色、琢型、比例、视觉重点或元素轮廓得到保留 |
| JD-8 | 批量设计符合品类配比，不把所有款式都做成同一品类 |
| JD-9 | 饱满高级珠宝预设不空、不薄、不散，视觉中心明确，副石聚拢有序 |
| JD-10 | 小轻金预设保持轻盈、小体量、少量宝石点缀，不变成厚重满钻 |
| JD-11 | 日式轻古董预设有细密工艺和轻古董气质，但结构仍真实可佩戴 |

失败处理：

- 结构错误：补充结构约束后重试。
- 款式重复：强化差异字段，按原 design_id 重试。
- 图好看但不真实：减少幻想元素，增加镶嵌、支撑、佩戴逻辑。
- 真实但不够高级：强化材质、光线、比例和主题钩子。
- 品类比例错误：重新生成设计矩阵，先锁定每个 design_id 的品类。
- 主石保留不足：在提示词中重复 `locked_reference`，明确主石颜色、琢型和比例不可改变。
- 构图过空或过散：提高 `design_density`，加入聚拢、饱满、围绕主石展开和边缘强调。
