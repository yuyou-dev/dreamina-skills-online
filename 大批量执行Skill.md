---
技能名称: 大批量执行Skill
技能描述: 即梦在线版大型项目规划、拆解、分批和并发执行管理技能。用于在调用具体生成类 skill 前，先把用户的大任务拆成可计数、可并发、可重试、可验收的任务清单。适合批量珠宝设计、批量电商素材、批量短视频分段、多个 SKU、多平台交付、N 款设计、N 条视频、N 组图片等大型任务。当用户说“大批量生成”“分批执行”“并发生成”“大型项目规划”“确保数量正确”“又快又好”“用两个 skill 先规划再执行”时使用。
---

技能内容：

本技能是**独立的大批量执行规划 skill**。它不能依赖外部 Markdown 文件；所有规则都写在本文件内。它的作用不是直接替代具体生成技能，而是先把大型项目拆成清晰任务树，再把每一批任务交给第二个具体生成 skill，例如 `珠宝电商素材Skill`、`JewelryDesignSkills`、`脱口秀视频Skill` 或其他即梦生成技能。

本技能的核心目标：

- 数量正确：用户要求 N 个，就规划 N 个终态产物或 N 个失败记录。
- 链条完整：每个下游任务必须知道依赖哪些上游结果。
- 分批合理：先小批验证，再扩大并发。
- 并发高效：同层级、无依赖的任务可以并发。
- 质量可控：每批都有质量门，失败项进入重试，不吞数量。
- 方便双 skill 测试：输出可以直接复制给第二个生成 skill 执行。

## 一、触发范围

### 1.1 应使用本技能的情况

- 用户要求一次生成很多内容，例如“设计 30 款”“做 10 个 SKU 的亚马逊九图”“生成 20 条短视频”“把 60 个分镜都做出来”。
- 用户要求“分批”“并发”“批量执行”“保证数量”“大型项目规划”。
- 用户想用双 skill 工作流：先用本技能规划，再用具体生成 skill 执行。
- 具体生成 skill 容易漏步骤、漏数量、链式调用中断，需要先建立任务清单。

### 1.2 不应使用本技能的情况

- 用户只要一个单图、一个短文案或一个很小的单步任务。
- 用户明确要求立即创作，不需要计划。
- 用户没有给目标产物，也无法判断任务类型；此时先收集目标。

## 二、交互阶段

### 步骤 1：信息提取

| 字段 | 必填 | 缺失时处理 |
|---|---|---|
| 项目目标 | 是 | 缺失时询问用户要生成什么，停止流程 |
| 数量 | 是 | 缺失时询问数量；如果用户说“一批/一套”，按对应预设推断并标注假设 |
| 目标产物类型 | 是 | 图片、视频、图文视频包、设计方案、分段素材等 |
| 具体生成 skill | 否 | 默认由任务类型推断，例如珠宝设计、电商素材、脱口秀视频 |
| 输入素材 | 否 | 有素材则列入依赖；无素材则规划母图/角色图/设计 brief |
| 预设 | 否 | 根据任务类型推断，例如 Amazon 9 图 + 10 秒视频 |
| 质量优先级 | 否 | 默认质量优先、速度兼顾 |
| 并发上限 | 否 | 图片默认 4-6，视频默认 2-4，最终拼接默认 1 |
| 批次大小 | 否 | 按总量自动计算 |

缺失必填字段时，调用 `generate_form_for_info_collection` 收集最小信息。可选字段不要阻塞流程。

### 步骤 2：识别任务类型

| 用户表达 | 任务类型 | 计数单位 |
|---|---|---|
| 设计 N 款、批量出款 | `design_batch` | `design_id` |
| N 个 SKU 做电商素材 | `ecommerce_sku_batch` | `sku_id × slot_id` |
| 亚马逊 9 图 + 视频 | `amazon_listing_batch` | 每 SKU 9 图 + 1 视频 + 1 文案 |
| 脱口秀视频分段 | `talk_show_segment_batch` | `segment_id + final_video` |
| N 条短视频 | `video_batch` | `video_id` |
| 多平台交付 | `multi_platform_batch` | `item_id × platform × slot_id` |

## 三、数量守恒规则

数量是承诺，不自动缩减。

### 3.1 核心规则

- 用户说 N 款，就必须规划 N 个稳定编号。
- 用户说 N 个 SKU，每个 SKU 都必须完整展开对应槽位。
- 用户说 9 图，必须是 9 个图片槽位，不是 1 张拼图替代 9 张，除非用户明确要拼图。
- 用户说完整视频，必须有最终拼接任务，不得只交付分段素材。
- 如果某项失败，必须记录失败项，不能从总数里消失。

### 3.2 汇总字段

每次规划和交付都必须输出：

```Markdown
## 数量汇总
- requested_count:
- planned_count:
- done_count:
- failed_count:
- missing_count:
- final_deliverable_count:
```

定义：

- `requested_count`：用户要求的目标数量。
- `planned_count`：任务清单展开后的计划数量。
- `done_count`：已完成数量。
- `failed_count`：失败且未修复数量。
- `missing_count`：计划中没有终态的数量。
- `final_deliverable_count`：最终可交付产物数量。

如果 `missing_count > 0`，不得说“全部完成”。

## 四、任务清单 Manifest

调用任何生成工具前，先输出 manifest。

### 4.1 Manifest 字段

| 字段 | 说明 |
|---|---|
| `job_id` | 稳定编号，不因失败重试而改变 |
| `group_id` | SKU、设计系列、视频项目或平台组 |
| `target` | 当前任务产物 |
| `tool` | 即梦工具或第二个 skill |
| `depends_on` | 上游依赖 |
| `input_refs` | 输入素材、脚本、母图、设计 brief |
| `batch_id` | 所属批次 |
| `parallel_group` | 可并发组 |
| `status` | `planned`、`running`、`done`、`failed`、`retrying` |
| `retry_count` | 重试次数 |
| `quality_gate` | 验收标准 |

### 4.2 Manifest 模板

```Markdown
## 执行 Manifest
| job_id | group_id | target | tool/skill | depends_on | batch_id | parallel_group | status | quality_gate |
|---|---|---|---|---|---|---|---|---|
```

## 五、依赖链规则

### 5.1 不可跳过的依赖

| 上游 | 下游 |
|---|---|
| 产品/角色/设计身份锁 | 图片生成、视频生成 |
| 产品母图 | 电商槽位图 |
| 脚本确认 | 脱口秀视频分段 |
| 主图/微距图/佩戴图 | 多模态视频 |
| 分段视频全部完成 | `video_editor` 最终拼接 |
| 第 1 批质量通过 | 扩大后续批次并发 |

### 5.2 链式调用写法

每个任务都要写“完成后下一步”：

```text
完成 job_id=SKU01-IMG01 后，把结果作为 @图片1 输入 SKU01-VID01。
完成 job_id=TS-S01 至 TS-S06 后，按顺序输入 TS-FINAL 的 video_editor。
完成 job_id=R01 主图后，再执行 R01-DETAIL 细节图，不并发跳过主图。
```

## 六、分批策略

### 6.1 默认批次大小

| 总任务量 | 建议批次 |
|---|---|
| 1-6 | 1 批 |
| 7-20 | 每批 4-6 |
| 21-60 | 每批 6-10 |
| 61 以上 | 每批 8-12，先做试运行批 |
| 视频分段 | 每批 2-4 |

### 6.2 首批验证

大批量任务必须先跑首批验证：

- 图片任务：首批 4-6 个。
- 视频任务：首批 1-2 个分段或 1 条短视频。
- 多 SKU 电商：首批 1 个 SKU 的完整链路，或 2-3 个槽位验证身份锁。

首批通过后再扩大并发。首批失败时，不继续盲目放大，要先修正提示词、身份锁、分段策略或质量门。

## 七、并发规则

### 7.1 可并发

- 同一批次中互不依赖的设计图。
- 同一 SKU 的多个独立电商图片槽位。
- 多个 SKU 的同类槽位。
- 脚本确认后的多个普通视频分段。
- 已经通过质量检查的图片超分或抠图。

### 7.2 不可并发

- 脚本未确认前生成脱口秀分段。
- 产品母图未通过前生成全部电商槽位。
- 角色关键帧未通过前生成所有视频分段。
- 视频分段未完成前执行最终拼接。
- 质量失败的图进入下游视频。

### 7.3 默认并发上限

| 类型 | 默认并发 | 原因 |
|---|---|---|
| 图片生成 | 4-6 | 快速，但仍能检查一致性 |
| 图片后处理 | 6-8 | 依赖少，可高并发 |
| 视频生成 | 2-4 | 更重、更容易漂移 |
| 脚本/文案 | 4-8 | 文本任务可高并发 |
| 最终拼接 | 1 | 必须按顺序合成 |

若平台限制并发，按平台限制执行，但 manifest 不变。

## 八、重试策略

失败项不能丢失。

### 8.1 重试规则

- 保留原 `job_id`。
- 记录失败原因。
- 增强对应约束。
- 重试次数默认最多 2 次，关键交付最多 3 次。
- 仍失败则标记 `failed`，进入最终 `failed_count`。

### 8.2 常见失败修复

| 失败 | 修复 |
|---|---|
| 角色/产品漂移 | 增加身份锁，减少动作复杂度，增加参考图 |
| 设计重复 | 强化差异字段，重写设计矩阵 |
| 数量漏项 | 回到 manifest，补齐缺失 job_id |
| 视频断裂 | 加上上段衔接、降低并发、必要时用首尾帧 |
| 电商槽位重复 | 重写槽位目的和构图 |
| 质量差但数量多 | 暂停扩大并发，先修正首批 |

## 九、双 skill 工作流

### 9.1 规划阶段：使用本技能

输出：

- 项目摘要。
- 数量汇总。
- 执行 manifest。
- 批次计划。
- 并发策略。
- 质量门。
- 交给第二个 skill 的执行包。

### 9.2 执行阶段：交给具体生成 skill

执行包模板：

```Markdown
# 交给生成 Skill 的执行包

## 使用技能
- skill:

## 当前批次
- batch_id:
- batch_size:
- concurrency:

## 本批任务
| job_id | target | input_refs | tool | depends_on | quality_gate |
|---|---|---|---|---|---|

## 执行要求
- 保留 job_id。
- 不改变数量。
- 完成后回填 status。
- 失败项保留原编号重试。
- 不把合集/预览当作单项交付。
```

### 9.3 回填阶段：继续使用本技能

当第二个 skill 返回结果后，本技能负责：

- 更新 `done/failed/missing`。
- 找出漏项。
- 生成下一批执行包。
- 决定是否扩大、降低或暂停并发。
- 最终汇总验收。

## 十、典型项目模板

### 10.1 批量珠宝设计

```Markdown
requested_count: N 个 design_id
第一层: 设计矩阵生成
第二层: 主设计图并发生成，4-6 首批，后续 6-10
第三层: 质量检查和失败重试
第四层: 可选细节图/电商素材移交
```

### 10.2 多 SKU 亚马逊电商素材

```Markdown
requested_count: SKU 数 × (9 图片 + 1 视频 + 1 文案)
第一层: 每个 SKU 的身份锁/母图
第二层: 每个 SKU 的 9 个图片槽位并发
第三层: 选择主图/微距/佩戴/礼盒作为视频参考
第四层: 每 SKU 生成视频
第五层: video_editor 整理最终视频
```

### 10.3 脱口秀视频

```Markdown
requested_count: confirmed_segment_count + 1 final_video
第一层: 角色分析
第二层: 脚本生成并等待用户确认
第三层: 关键帧生成
第四层: 分段视频 2-4 并发
第五层: video_editor 拼接完整视频
```
