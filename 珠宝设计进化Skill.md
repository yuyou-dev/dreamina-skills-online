---
技能名称: 珠宝设计进化
技能描述: 选择题驱动的单轮珠宝设计进化技能。用户上传一张底座珠宝图、草图、元素图，或用文字描述一个初始珠宝雏形后，本技能通过单选/多选问题收集结构保留、进化强度、元素主题、形态语言、审美风格、材质工艺和输出数量，先生成进化设计矩阵，再批量生成 4 或 8 张可比较的珠宝进化候选图。本技能默认一轮结束，不自动触发第二轮；如用户要继续，应基于选中的候选图重新发起一次进化。当用户说“做珠宝设计进化”“给这款珠宝做几种进化方向”“用选择题帮我迭代款式”“先出 4 个方向我再选”“把这张图继续演化”时使用。
---

技能内容：

分为**交互阶段**和**执行阶段**。交互阶段用选择题收集进化变量；执行阶段分析底座、生成进化矩阵、批量调用图片工具、质量检查、输出可选择的候选方案和下一轮启动包。本技能不在同一次运行中自动进入第二轮进化。

核心链路：

```text
底座输入 → 选择题表单 → 底座身份锁 → 进化矩阵 → 批量图片生成 → 质量检查 → 候选评审 → 单轮交付
```

默认交付：

- 4 个进化方向候选图。
- 每个方向 1 张主图 + 结构说明 + 进化理由。
- 每个候选都给出“可作为下一轮底座”的启动 brief，但不自动调用第二轮。

模型默认：

- 图片类任务默认使用 4.7 模型的 2k 版。
- 视频类任务默认使用 `seedance2.0fast_vip`。
- 用户明确指定图片或视频模型版本时，按用户指定执行，不再套用对应默认模型。

## 一、触发范围

### 1.1 应使用本技能的情况

- 用户想在已有珠宝图、草图、裸石图、元素图基础上做多方向演化。
- 用户希望通过选择题决定设计方向，而不是一次性写完整 prompt。
- 用户要求“先给我 4 个方向 / 8 个方向，我先看一轮”。
- 用户需要保留原款结构，同时尝试不同元素、材质、风格或工艺。
- 用户想做系列探索、款式创改、竞品改造、文化元素转译、设计分支评审。

### 1.2 应移交其他技能的情况

- 用户只要单款高质量原创珠宝设计，不需要选择题或进化矩阵 → 使用 `JewelryDesignSkills`。
- 用户已经确定款式，要做亚马逊九图、上架图、详情页、短视频和文案 → 使用 `珠宝电商素材Skill`。
- 用户要一次规划几十款或多个 SKU 的大项目 → 先使用 `大批量执行Skill` 拆解，再交给本技能或其他生成技能。

## 二、工具映射

| 工具名 | 本技能用途 | 链式调用提示 |
|---|---|---|
| `generate_form_for_info_collection` | 用选择题收集进化变量 | 只在初始进化问题、用户要改选项或缺少必填字段时使用；表单完成后进入矩阵 |
| `creation_agent_search` | 可选：查询非侵权文化元素、珠宝风格、工艺参考 | 搜索后转成原创约束，不直接复制品牌款 |
| `text2image` / `text2image_v3` | 无底座图时生成初始珠宝底座图 | 默认 4.7 模型 2k 版；生成后作为底座进入进化 |
| `image2image` / `image2image_v3` | 基于底座图、草图、元素图或用户重新上传的选中结果生成本轮进化候选 | 默认 4.7 模型 2k 版；每个候选都重复底座身份锁和进化变量 |
| `foreground_segmentation` | 抽取候选主体，用于对比板或后续电商素材 | 可选，候选确认后再使用 |
| `image_super_resolution` | 放大优秀候选图或最终选中图 | 结构正确后再超分 |
| `image2video` / `multi_modal2video` | 只在用户要求进化过程展示短片时使用 | 默认 `seedance2.0fast_vip`；图片结果先完成再进视频 |
| `video_editor` | 拼接进化过程视频或候选展示视频 | 视频流程最后调用 |

在线版适配：

- 图片工具如存在 v3 版本，优先使用 `text2image_v3` / `image2image_v3`。
- 图片生成默认选择 4.7 模型 2k 版。
- 运行时优先使用“产品设计”“商业摄影”“参考图改造”“高清细节”类在线预设。
- 不写本地 CLI 模型名、账号命令或任务查询命令。

## 三、交互阶段

### 步骤 1：信息提取

| 字段 | 必填 | 缺失时处理 |
|---|---|---|
| 底座图 / 草图 / 元素图 / 文字雏形 | 是 | 缺图且无文字雏形时，调用 `generate_form_for_info_collection` 提示上传或描述 |
| 目标珠宝类型 | 否 | 可从底座图判断；无法判断时默认“保留底座类型” |
| 进化轮次 | 否 | 默认第 1 轮；只有用户重新发起进化时才作为手动记录 |
| 输出数量 | 否 | 默认 4 个候选；可选 8 个候选 |
| 结构保留度 | 否 | 默认“中：保留轮廓，允许局部变化” |
| 进化强度 | 否 | 默认“平衡：明显变化但保留识别度” |
| 融合策略 | 否 | 默认“纹样转译” |
| 元素主题 | 否 | 默认由底座图和用户描述推断；也可让用户多选 |
| 形态语言 | 否 | 默认“原型轮廓 + 流线曲面” |
| 审美风格 | 否 | 默认“高级珠宝 + 当代极简” |
| 材质工艺 | 否 | 默认保留底座主材质，并加入钻石或贵金属细节 |
| 参考图 | 否 | 有参考图则只作为风格、材质、工艺或细节参考，不抢底座轮廓 |
| 参考图角色 | 否 | 默认“风格参考”；多图时按图片内容推断 |
| 用户补充方向 | 否 | 默认无；有则写入矩阵和每个候选提示词 |
| 图片模型 | 否 | 默认 4.7 模型的 2k 版；用户指定模型版本时按指定 |

缺少必填字段时才中断流程。可选字段用默认值，不要连续追问。

### 步骤 2：选择题表单

当用户没有明确给出进化变量时，调用 `generate_form_for_info_collection` 生成选择题表单。表单字段如下：

| 题目 | 类型 | 选项 | 默认 |
|---|---|---|---|
| 本轮输出几个进化方向？ | 单选 | 4 个 / 8 个 | 4 个 |
| 希望多大程度保留原款结构？ | 单选 | 高：尽量保留结构与比例 / 中：保留轮廓，允许局部变化 / 低：只保留家族相似性 | 中 |
| 本轮进化强度？ | 单选 | 微调 / 平衡 / 重塑 | 平衡 |
| 元素如何与底座融合？ | 单选 | 保形嫁接 / 纹样转译 / 结构重组 | 纹样转译 |
| 想注入哪些元素主题？ | 多选 | 藤蔓花叶 / 花朵绽放 / 羽翼生灵 / 昆虫宝石 / 星月天象 / 水波海洋 / 几何徽章 / 建筑拱廊 / 护符符号 / 东方云纹 / 自定义 | 默认从底座推断 2-3 个 |
| 形态语言偏好？ | 多选 | 原型轮廓 / 流线曲面 / 几何秩序 / 镂空花丝 / 浮雕层叠 / 仿生有机 / 模块串联 / 立体雕塑 | 原型轮廓 + 流线曲面 |
| 审美风格偏好？ | 多选 | 当代极简 / 新中式雅奢 / 东方宫廷 / Art Deco / 高级珠宝 / 自然主义 / 婚嫁典雅 / 年轻潮奢 | 高级珠宝 + 当代极简 |
| 材质工艺偏好？ | 多选 | 24K 纯金 / 珍珠母贝 / 彩色宝石 / 冷珐琅 / 热珐琅 / 钻石 / 双色贵金属 | 保留底座主材质 + 钻石 |
| 参考图主要用于什么？ | 单选 | 风格参考 / 材质参考 / 工艺参考 / 细节参考 / 氛围参考 | 风格参考 |

表单完成后，必须继续进入“底座身份锁”和“进化矩阵”，不要停在表单摘要。

### 步骤 3：确认与跳过规则

- 用户已经明确所有关键方向 → 不再调用表单，直接生成进化矩阵。
- 用户只给底座图，没有方向 → 自动调用选择题表单。
- 用户说“你默认帮我选” → 使用默认 4 候选、结构保留度中、进化强度平衡、融合策略纹样转译。
- 用户说“大胆一点” → 进化强度设为重塑，结构保留度可降为低或中。
- 用户说“别改太多” → 结构保留度设为高，进化强度设为微调。
- 用户说“基于上一轮某个结果继续” → 要求用户重新上传或明确引用该候选图；本次运行把它当作新的底座处理，不尝试承接上一轮表单状态。

## 四、执行阶段

### 步骤 4：底座身份锁

执行任何生成前，先建立底座身份锁。

底座身份锁字段：

| 字段 | 说明 |
|---|---|
| `base_id` | 底座编号，如 `BASE-R1`、`BASE-R2` |
| `base_type` | 戒指、项链、耳环、手链、吊坠、胸针、套装等 |
| `base_silhouette` | 原始轮廓、比例、体量和视觉中心 |
| `base_structure` | 戒臂、链条、吊环、耳针、扣具、铰链、镶口等真实结构 |
| `base_material` | 金属、宝石、珐琅、珍珠、母贝等 |
| `locked_parts` | 必须保留的结构、主石、比例、佩戴逻辑 |
| `editable_parts` | 可进化的局部，如边缘、纹样、镶嵌、吊坠轮廓、戒肩 |
| `reference_roles` | 参考图角色：风格、材质、工艺、细节、氛围 |

底座图不是普通参考图。底座图负责结构，参考图负责语言。

如果没有底座图但有文字雏形：

1. 调用 `text2image_v3`，默认 4.7 模型 2k 版生成初始底座图。
2. 检查底座图是否清晰、结构可佩戴。
3. 合格后把生成图作为 `BASE-R1` 进入进化矩阵。

### 步骤 5：进化变量解释

#### 5.1 结构保留度

| 选项 | 执行约束 |
|---|---|
| 高 | 尽量保持底座轮廓、比例、主石位置和佩戴结构，只改装饰、纹样、边缘和镶嵌 |
| 中 | 保留底座家族轮廓和主结构，允许局部体量、边缘、支撑结构和视觉中心变化 |
| 低 | 只保留底座识别基因，允许更大胆重组，但仍必须是可佩戴珠宝 |

如果用户提供局部蒙版或明确局部区域，只修改指定区域，其他区域保持不变。

#### 5.2 进化强度

| 选项 | 执行约束 |
|---|---|
| 微调 | 适合商业款优化，只增强比例、边缘、镶嵌和材质细节 |
| 平衡 | 适合新系列探索，保留识别度，同时让元素主题清晰进入设计 |
| 重塑 | 适合概念方向探索，允许轮廓和结构重组，但不能变成不可佩戴装置 |

#### 5.3 融合策略

| 选项 | 执行约束 |
|---|---|
| 保形嫁接 | 保留原结构，把元素作为局部结构、宝石排列、边缘装饰植入 |
| 纹样转译 | 把元素转化为纹样、镂空、浮雕、肌理、爪镶节奏或宝石排列 |
| 结构重组 | 把元素变成新的结构逻辑，重组局部轮廓、视觉重心或连接方式 |

#### 5.4 元素主题库

| 元素 | 适合转译方式 |
|---|---|
| 藤蔓花叶 | 曲线戒臂、缠绕链节、叶片边缘、小钻露珠 |
| 花朵绽放 | 花簇镶嵌、中心主石、花瓣副石、层叠轮廓 |
| 羽翼生灵 | 羽片分层、轻盈开合、边缘渐变钻、耳饰垂坠 |
| 昆虫宝石 | 对称翅面、彩宝身体、细密镶嵌、胸针或戒面 |
| 星月天象 | 星点钻石、月弧轮廓、深色宝石、渐变光晕 |
| 水波海洋 | 流线曲面、波纹镶嵌、蓝绿宝石、贝壳光泽 |
| 几何徽章 | 对称结构、阶梯形、Art Deco、硬朗金属边 |
| 建筑拱廊 | 拱形结构、透空镂空、层级支撑、宫廷秩序 |
| 护符符号 | 中心护符、圆牌、平安扣、边缘纹样、寓意符号 |
| 东方云纹 | 卷云线条、祥云边缘、新中式雅奢、金属浮雕 |

#### 5.5 形态语言库

- 原型轮廓：保留底座外形与比例。
- 流线曲面：柔和曲线、人体工学体量、自然流动。
- 几何秩序：对称、阶梯、几何切面、比例清晰。
- 镂空花丝：透空结构、精细线条、负空间可读。
- 浮雕层叠：前后层级、浅浮雕、边缘细节。
- 仿生有机：自然生长感、非机械曲线、生命节奏。
- 模块串联：重复单元、链节、可延展系列结构。
- 立体雕塑：强侧面、体积感、可从多角度观看。

#### 5.6 审美风格库

- 当代极简：克制、干净、少量高质量细节。
- 新中式雅奢：东方文化符号现代化，避免古董化堆砌。
- 东方宫廷：仪式感、对称、华丽但结构可信。
- Art Deco：阶梯几何、清晰对称、强边框。
- 高级珠宝：精密镶嵌、收藏级比例、宝石和金属高质感。
- 自然主义：植物、动物、自然纹理的真实细节。
- 婚嫁典雅：柔和、浪漫、稳定佩戴、适合礼赠。
- 年轻潮奢：更轻快、时髦、有文化身份但不廉价。

#### 5.7 材质工艺库

- 24K 纯金：厚重、温润、哑光或细砂质感，避免塑料黄。
- 珍珠母贝：虹彩、贝壳层次、柔和反光。
- 彩色宝石：蓝宝石、祖母绿、红宝石、粉色蓝宝石、碧玺等。
- 冷珐琅：清晰色块、现代图案、表面平整。
- 热珐琅：更深的釉色、传统工艺感、边界精致。
- 钻石：密镶、边缘强调、主石光晕、小颗露珠。
- 双色贵金属：白金 + 黄金、玫瑰金 + 白金、冷暖对比。

### 步骤 6：进化矩阵

在调用图片工具前，先生成进化矩阵。矩阵数量必须等于用户选择的输出数量。

默认 4 个候选应覆盖不同设计逻辑：

| 候选 | 默认方向 | 目标 |
|---|---|---|
| A | 保守商业化 | 最容易落地销售，结构变化最小 |
| B | 元素明确化 | 元素主题更清楚，适合系列主款 |
| C | 工艺材质化 | 重点探索材质、镶嵌和表面工艺 |
| D | 大胆概念化 | 更强视觉记忆点，但仍可佩戴 |

8 个候选在 A-D 基础上扩展：

| 候选 | 默认方向 | 目标 |
|---|---|---|
| E | 高级珠宝化 | 更饱满、更华丽、更收藏级 |
| F | 日常轻量化 | 更轻盈、更适合日常佩戴 |
| G | 文化符号化 | 强化东方、护符、吉祥或叙事主题 |
| H | 系列延展化 | 更适合发展成耳饰、项链、戒指套组 |

矩阵字段：

| 字段 | 说明 |
|---|---|
| `evolution_id` | 稳定编号，如 `EVO-R1-A` |
| `round` | 进化轮次 |
| `candidate_label` | A-H |
| `base_id` | 使用的底座 |
| `target_type` | 候选珠宝类型 |
| `structure_fidelity` | 高 / 中 / 低 |
| `intensity` | 微调 / 平衡 / 重塑 |
| `fusion_strategy` | 保形嫁接 / 纹样转译 / 结构重组 |
| `element_theme` | 元素主题 |
| `morphology` | 形态语言 |
| `style` | 审美风格 |
| `material` | 材质工艺 |
| `locked_parts` | 必须保留 |
| `editable_parts` | 本候选重点改动 |
| `difference` | 与其他候选的差异 |
| `prompt` | 图片提示词 |
| `model` | 图片模型；默认 4.7 模型 2k 版 |
| `status` | planned / done / failed / retrying |

批量生成前输出 manifest：

```Markdown
| job_id | round | candidate | target | tool | model | depends_on | status | quality_gate |
|---|---|---|---|---|---|---|---|---|
| EVO-R1-BASE | 1 | BASE | 底座身份锁 | 无（技能内部分析） |  | 用户输入 | done | JE-B1 |
| EVO-R1-A | 1 | A | 保守商业化候选图 | image2image_v3 | 4.7 模型 2k 版 | EVO-R1-BASE | planned | JE-I1/JE-I4 |
| EVO-R1-B | 1 | B | 元素明确化候选图 | image2image_v3 | 4.7 模型 2k 版 | EVO-R1-BASE | planned | JE-I1/JE-I4 |
| EVO-R1-C | 1 | C | 工艺材质化候选图 | image2image_v3 | 4.7 模型 2k 版 | EVO-R1-BASE | planned | JE-I1/JE-I5 |
| EVO-R1-D | 1 | D | 大胆概念化候选图 | image2image_v3 | 4.7 模型 2k 版 | EVO-R1-BASE | planned | JE-I1/JE-I6 |
```

数量规则：

- `requested_count = 用户选择的候选数`。
- `planned_count` 必须等于候选数。
- 每个候选必须有稳定 `evolution_id`。
- 失败项保留原编号重试，不用新编号替换。
- 不能用 1 张合集图代替 4 或 8 张单独候选图，除非用户明确只要合集预览。

### 步骤 8：图片生成

调用规则：

- 有底座图 / 用户重新上传的选中结果图 → 使用 `image2image_v3`，默认 4.7 模型 2k 版。
- 只有文字雏形 → 先使用 `text2image_v3` 生成底座，再使用 `image2image_v3` 进化。
- 有参考图 → 底座图作为 `@图片1`，参考图作为 `@图片2` 起，明确参考图角色。
- 有局部蒙版或用户指定区域 → 蒙版作为最后一张输入图，提示只修改高亮区域。

同批候选可并发生成：

- 4 个候选默认同批生成。
- 8 个候选可分两批，每批 4 个。
- 如果前 2 个候选明显漂移，暂停后续并发，强化底座身份锁再重试。

### 步骤 10：候选评审与单轮收口

候选生成后，必须把结果组织成可比较的候选评审表。本技能在这里完成本轮交付，不自动调用第二轮表单，也不等待后续按钮或选项来承接新一轮状态。

评审格式：

```Markdown
## 本轮候选评审

| 选项 | 方向 | 推荐用途 | 可继续强化点 |
|---|---|---|---|
| A | {A_title} | {A_use_case} | {A_next_hint} |
| B | {B_title} | {B_use_case} | {B_next_hint} |
| C | {C_title} | {C_use_case} | {C_next_hint} |
| D | {D_title} | {D_use_case} | {D_next_hint} |
```

收口规则：

- 本轮输出到候选评审、收尾操作和下一轮启动包为止。
- 不承诺自动接收 A/B/C/D 选择后继续生成。
- 如果用户要继续，把本轮选中的候选图和对应启动 brief 作为下一次独立运行的输入。
- 如果用户要高清确认，对选中图调用 `image_super_resolution`，这仍属于本轮收尾，不是第二轮进化。
- 如果用户要电商素材，输出产品身份锁，移交 `珠宝电商素材Skill`。

### 步骤 11：下一轮启动包

下一轮启动包只用于用户手动复制或重新发起，不在本轮自动执行。

每个候选都输出一个启动包：

```Markdown
## 下一轮启动包

### 候选 A
- 新底座: 候选 A 图片
- 建议指令: 基于候选 A 继续做珠宝设计进化，保留 {A_locked_parts}，重点强化 {A_next_hint}。
- 建议结构保留度:
- 建议进化强度:
- 建议融合策略:
- 建议保留变量:
- 建议替换变量:
```

启动包规则：

- 下一轮必须由用户重新发起，或重新上传 / 引用选中的候选图。
- 下一轮不依赖本轮的表单交互状态。
- 启动包必须写清保留变量和建议替换变量，避免用户重新描述全部信息。
- 如果用户说“混合 A+C”，输出一个混合启动包，不自动执行混合进化。

## 五、质量检查

### 5.1 底座检查

| 编号 | 标准 |
|---|---|
| JE-B1 | 底座类型、轮廓、主石、结构和佩戴逻辑已经识别 |
| JE-B2 | 必须保留与可改动区域清楚 |
| JE-B3 | 参考图角色没有和底座图混淆 |

### 5.2 进化矩阵检查

| 编号 | 标准 |
|---|---|
| JE-M1 | 候选数量等于用户选择数量 |
| JE-M2 | 每个候选有稳定 `evolution_id` |
| JE-M3 | 候选之间差异清楚，不只是换颜色 |
| JE-M4 | 每个候选都包含元素、形态、风格、材质和融合策略 |

### 5.3 图片质量检查

| 编号 | 标准 |
|---|---|
| JE-I1 | 底座关键结构或家族相似性符合结构保留度 |
| JE-I2 | 元素主题已经珠宝化，不是平面贴图 |
| JE-I3 | 材质、宝石、镶嵌和佩戴结构真实可信 |
| JE-I4 | 候选之间有明显设计差异 |
| JE-I5 | 画面干净，无文字、logo、价格、证书或水印 |
| JE-I6 | 大胆方向仍然可佩戴，不是概念装置 |

失败处理：

- 底座漂移：提高结构保留度，重复底座身份锁，减少进化强度。
- 变化太小：提高进化强度，强化差异字段和元素主题。
- 元素贴图感强：改为纹样、镂空、浮雕、镶嵌、结构转译。
- 结构不可佩戴：补充戒臂、链条、耳针、吊环、扣具、镶口等结构约束。
- 候选重复：重写进化矩阵，锁定 A-D 的不同定位。
- 图像质量不足：结构正确后调用 `image_super_resolution`。

## 七、触发示例

- “上传这张戒指，帮我用选择题做珠宝设计进化，先出 4 个方向。”

- “不要改太多，只做商业化微调。”

## 八、禁止项

- 不把底座图当成普通风格参考而忽略结构。
- 不把参考图的整体轮廓复制到新设计中。
- 不把 4 个候选做成同一款的轻微颜色变化。
- 不生成不可佩戴、无支撑、漂浮宝石或结构断裂的珠宝。
- 不编造品牌授权、证书、克拉数、产地、价格或库存。
- 不在图片中加入文字、水印、logo、价格牌或复杂背景。
